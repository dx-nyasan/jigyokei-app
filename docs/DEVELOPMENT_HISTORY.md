## 2. フェーズ別推移分析

### Phase 0-1: 基盤構築期（Initial → Core）
- **理念**: 暗黙的。AIチャットで申請書を作れるようにする。
- **主要物**: `app_hybrid.py`, `schemas.py`, `chat_manager.py`

### Phase 2-3: 可視化の確立（Dashboard → Severity）
- **理念**: 「透明性」「導く」。何が足りないかを明確に伝える。
- **主要物**: スコアリングロジック, Critical/Warning分類, 下書きシートExport

### Phase 4-5: ユーザー多様化（Mobile → Manuals）
- **理念**: 「寄り添い」「引き出し」。三者視点（経営者/従業員/職員）の尊重。
- **主要物**: モバイル対応, Agenticデータ抽出, 3ロール別マニュアル

### Phase 6: アーキテクチャ革新（Certification-Driven）
- **理念**: 「認定駆動」「持続性」。要件をコードと1:1対応させ、RAGで知識更新。
- **主要物**: `certification_requirements.py`, `audit_agent.py`, `manual_rag.py`

### Phase 7: 知識の完全統合（Skill-Driven Protocol）
- **理念**: 「自律開発」「不文律の形式知化」。分散した全ての知識を「Skill」としてAntigravityに宿す。
- **主要物**: `core-protocol`, `jigyokei-domain-expert`, 階層型Skills体系

## 3. 今後の哲学記録ルール
このドキュメントは `technical-writer` Skill によって、各フェーズ完了時に更新されます。
単なる「機能追加」ではなく、その裏側にある「なぜその設計にしたのか」「どのような価値を追求したのか」という**哲学的な進化**を記録し続けること。
