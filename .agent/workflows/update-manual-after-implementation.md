# 実装ワークフロー（TDD + Docs + マニュアル更新）

このワークフローは、LangGraph統合プロジェクトの実装時に適用されます。

## ルール 1: テスト駆動開発（TDD）

> **必須**: 実装前に必ず失敗するテストを書くこと

1. **Red（失敗）**: まず失敗するテストコードを書く
2. **Green（成功）**: テストが通る最小限の実装を行う
3. **Refactor（整理）**: コードを整理し、テストが通ることを確認

## ルール 2: ドキュメント同時生成

> **必須**: コード生成と同時にドキュメントを生成すること

1. **関数/クラスには必ずdocstring**を付与
2. **新規ファイル作成時**は先頭にモジュールdocstringを記載
3. **複雑なロジック**にはインラインコメントを追加
4. **APIの使い方**は `user_manual.md` に即時反映

## ルール 3: マニュアル更新

実装完了時に以下を更新:
- `user_manual.md` - 新機能の使い方
- `walkthrough.md` - 実装内容と検証結果
- `task.md` - チェックリスト更新

## 実行手順

1. 失敗するテストを書く (TDD Red)
2. 実装 + docstring同時記載 (TDD Green + Docs)
3. テスト成功を確認
4. マニュアル類を更新
5. task.mdのチェックリストを更新

## 関連ファイル

- artifacts: `user_manual.md`, `walkthrough.md`, `implementation_plan.md`
