# core-protocol Skill (The Constitution)

このプロジェクトのすべての開発作業において、最優先で遵守すべき「憲法」です。
他のすべての専門Skillは、このProtocolを継承し、前提として動作します。

## 1. テスト駆動開発（TDD）の徹底
いかなる変更も、以下のサイクルを外れてはなりません。
- **Red**: 失敗するテストコードを `tests/` 配下に作成する。
- **Green**: テストを通過させる最小限の実装を行う。
- **Refactor**: プロトコルを維持したまま、コードの品質を向上させる。

## 2. ドキュメントの同時生成と同期
「コードがある場所には常にドキュメントがある」状態を維持します。
- **コード内**: Googleスタイルのdocstring、複雑なロジックへのコメント。
- **外部**: `user_manual.md`, `walkthrough.md`, `task.md` への即時反映。

## 3. 品質基準
- **命名**: `GEMINI.md` の規定に従い、意図が明確な命名を行う。
- **疎結合**: 各コンポーネントは独立してテスト可能であること。

## 他のSkillとの連携ルール
専門Skill（task-planner, code-reviewer等）を使用する際は、その専門手順を実行する前に、必ずこの `core-protocol` の要件が満たされているかを確認しなければなりません。
